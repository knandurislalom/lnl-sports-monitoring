## Common Git Commands

### Daily Workflow
```bash
# Check status and pull latest changes
git status
git pull origin main

# Create feature branch
git checkout -b feature/component-name

# Stage and commit changes
git add .
git commit -m "type(scope): description"

# Push feature branch
git push -u origin feature/component-name

# Merge back to main (after PR approval)
git checkout main
git pull origin main
git merge feature/component-name
git push origin main
```

### Commit Message Format
```
type(scope): description

Types: feat, fix, docs, style, refactor, test, chore
Scope: component, module, or area affected
Examples:
- feat(auth): add user authentication
- fix(api): resolve data fetching issue
- docs(readme): update installation instructions
```

## Troubleshooting
```bash
# Authentication issues
gh auth login
# or
git config --global credential.helper store

# Permission denied
# Check SSH key: ssh -T git@github.com
# Or use HTTPS with token

# Sync fork with upstream
git remote add upstream https://github.com/knandurislalom/lnl-sports-monitoring.git
git fetch upstream
git checkout main
git merge upstream/main
```
